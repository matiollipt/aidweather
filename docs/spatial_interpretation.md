# Spatial Interpretation Guide — `aidweather`

Understanding the spatial structure and resolution of NASA POWER parameters is essential for rigorous environmental, agricultural, and climatological analyses.

---

## 1. Grid Cell Averages vs. Weather Station Observations

NASA POWER does not provide point observations from physical ground-based weather stations. Instead, weather parameters are generated by global atmospheric reanalysis models and satellite observations.

When you request data for a specific coordinate (e.g. `-23.55° lat, -46.63° lon`):
1. NASA POWER locates the containing source product grid cell for that coordinate.
2. The returned values represent the spatial average across that entire grid cell.
3. Multiple coordinates situated within the same source grid cell will return identical weather values.

---

## 2. Non-Uniform Parameter Resolutions

NASA POWER integrates datasets from multiple satellite missions and atmospheric reanalyses, each operating at its native spatial resolution:

| Parameter Family | Primary Source Product | Native Grid | Spatial Cell Size (~Eq.) |
| :--- | :--- | :--- | :--- |
| Meteorological (`T2M`, `PRECTOTCORR`, `RH2M`, `WS10M`, `PS`, etc.) | NASA GMAO MERRA-2 / GEOS-IT | 0.50° Lat × 0.625° Lon | ~55.5 km × 69.4 km |
| Solar Radiation (`ALLSKY_SFC_SW_DWN`, `ALLSKY_SFC_PAR_TOT`) | NASA LaRC CERES SYN1deg / SRB | 1.00° Lat × 1.00° Lon | ~111.1 km × 111.1 km |

> [!IMPORTANT]
> Do not assume all NASA POWER parameters share a uniform grid. When combining temperature (`T2M`) and solar radiation (`ALLSKY_SFC_SW_DWN`) in a single point query, the solar values reflect a broader 1.0° cell than the 0.5° × 0.625° meteorological cell.

---

## 3. Spatial Transect Sampling & Clamping Policy

When sampling along a linear 1D transect path, specifying sample points closer together than the native grid cell produces redundant requests that yield duplicate values.

To avoid unnecessary server hits and prevent misleading spatial sampling:
- `aidweather` automatically determines the minimum effective transect spacing based on the finest native resolution among the requested parameters.
- For MERRA-2 meteorological parameters (`T2M`), the minimum effective spacing threshold is ~0.5° (~55.5 km).
- For CERES solar radiation parameters (`ALLSKY_SFC_SW_DWN`), the minimum effective spacing threshold is ~1.0° (~111.1 km).
- Requests specifying spacing below these limits emit transparent log warnings and are clamped to `max_allowed` distinct points.

---

## 4. Regional Bounding-Box Boundaries

The regional endpoint returns time series for all grid cell centers falling within a specified 2D bounding box:
- Bounding boundaries are inclusive (`latitude-min`, `latitude-max`, `longitude-min`, `longitude-max`).
- Maximum allowable spatial span per request: 4.5° latitude × 4.5° longitude.
- Requests crossing the antimeridian (180° longitude) or specifying inverted boundaries (`lat_min >= lat_max`) are rejected with clear `ValueError` messages.
